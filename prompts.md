# 开发提示词清单

## 结构化开发提示词（8段）

### 【提示词 1/8】项目总览与技术栈定义
- 目标：搭建宪法序言交互式教学网页框架
- 结构：首页、序言页、版本页、闯关页、共创页
- 视觉约束：深红色主色、金色辅助色、去AI化（无渐变、非对称、双字体）
- 验收：文件结构完整、无控制台错误

### 【提示词 2/8】CSS基础层与设计规范
- 目标：建立设计系统基础
- 结构：CSS变量、基础样式、工具类
- 约束：颜色变量、字号层级、4px网格
- 验收：变量完整、字体加载正确

### 【提示词 3/8】公共组件开发
- 目标：导航栏、按钮、页脚
- 结构：导航（汉堡菜单）、按钮系统、页脚
- 约束：半透明导航、无过度动画
- 验收：响应式正常、组件可复用

### 【提示词 4/8】首页开发
- 目标：模块入口展示
- 结构：Hero区（非对称）、模块入口区、CTA区
- 约束：非对称布局、无固定模板
- 验收：跳转正确、响应式正常

### 【提示词 5/8】宪法序言展示页
- 目标：序言完整展示
- 结构：标题区、正文区、关键词高亮、滚动动画
- 约束：左对齐、最大宽度720px
- 验收：内容正确、动画流畅

### 【提示词 6/8】版本迭代展示页
- 目标：展示版本变化
- 结构：时间轴、版本展示、对比区
- 约束：差异高亮、无过度动画
- 验收：切换正确、对比清晰

### 【提示词 7/8】知识闯关页
- 目标：问答游戏
- 结构：首页、答题区、反馈区、结果页
- 约束：进度条、计时器、选择题
- 验收：题目随机、分数正确

### 【提示词 8/8】共创宪法序言页
- 目标：用户续写共创
- 结构：输入区、徽章、全班墙、生成长图
- 约束：关键词匹配、点赞排序
- 验收：徽章触发正确、长图生成成功

## 💡 开发经验总结

### 成功的经验
- 分模块开发，逐个突破
- 使用 Git 版本控制，保留历史版本
- 单文件版本便于分发，是一个很好的备份方案

### 踩过的坑
- JSON 数据加载会有跨域问题，需要服务器环境
- 印章位置调整了多次，最终选择了放在标题旁边
- 字体显示问题，特别是繁体书法字体的兼容性

### 下次可以改进的地方
- 提前考虑数据共享问题
- 更早引入响应式设计
- 多做用户测试
